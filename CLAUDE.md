# Chalssak 프로젝트 지침

## 프로젝트 개요
- Google NotebookLM 클론 코딩 (풀스택)
- 자세한 스펙은 Linear Chalssak 프로젝트 참조

## 팀 구조
- **나**: 팀장 겸 개발자 겸 팀원 (원맨팀)
- **Claude Code**: 실제 개발 담당
- **PM**: 사람, 프로젝트 관리 및 바이브성 코칭

## 개발 방식
- **바이브 코딩**: Claude Code가 개발, 나는 검수 위주
- PM이 바이브성을 키우길 원함
- 예상 피드백: "이 부분은 자동화할 수 있는데 네가 직접했어"
- 자동화 가능한 부분은 최대한 Claude Code에게 위임할 것

## PM 지시사항
1. 오픈소스(오픈노트북 등) 참고하지 말고 바닥부터 구현할 것
2. Linear 사용법 숙지할 것
3. 프로젝트 완성도보다 **에이전트와 효율적으로 협업하는 방법 학습**에 초점
4. 작업 후 회고: 에이전트와 더 효율적으로 일할 수 있는 방법이 있었는지 생각해볼 것

## Claude Code 행동 지침
- 적극적으로 자동화 제안하기
- 반복 작업은 스스로 처리하기
- 사용자가 직접 할 필요 없는 작업은 대신 수행하기
- Linear 연동 적극 활용하기

## 테스트 원칙
- **매 기능별로 테스트 코드 작성 필수**
- 테스트가 모두 통과해야 다음 기능으로 넘어감
- `pytest -v` 실행하여 결과 확인
- 기존 테스트가 깨지지 않았는지 반드시 검증
- Mock 기반 단위 테스트 + 통합 테스트 병행
